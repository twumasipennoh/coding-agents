# Known Failures (Per-Project)

Project-specific failure rules extracted from past bug fixes. For cross-project patterns, see `~/.claude/known-failures.md` (global).

**Read by:** requirements-clarifier (Phase 3), feature-creator, fix-advocate, test-creator, pattern-enforcer (backstop cross-check).

**Written by:** /fix and /patch capture step (semi-auto, user-approved). Rules that appear in 2+ projects get promoted to global by /memory-review.

## Format

Each rule has:
- **Domain tag** `[domain]` — technology/area
- **Category tag** `[CATEGORY]` — WIRING, LOGIC, STATE, CONFIG, UI, PLATFORM, TYPE
- **Trigger** — when this rule is relevant
- **Failure mode** — what breaks and how
- **Prevention** — what to do instead
